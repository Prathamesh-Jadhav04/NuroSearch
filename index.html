How It Works

Embedding: Question → Ollama nomic-embed-text → 768D vector

Vector Search: HNSW graph traversal finds nearest neighbors in O(log N) time

Context Assembly: Top-k chunks retrieved and assembled into prompt

LLM Answer: qwen2.5:0.5b generates answer from context only

Project Highlights
  • Engineered a production-grade vector database from scratch with HNSW, KD-Tree, and Brute Force search algorithms
  • Implemented hybrid search combining semantic (vector) + lexical (BM25) retrieval for higher accuracy
  • Built a full RAG pipeline with chunking, embedding, context retrieval, and LLM-powered Q&A
  • Designed a real-time 3D PCA visualization for semantic space clustering analysis
  • Integrated SQLite persistence, Redis caching, and Ollama local LLM for end-to-end data flow
  • Delivered a complete REST API with CRUD operations, benchmarking, and streaming responses
  • Optimized search performance with MD5 cache keys, lock-scope reduction, and O(k) hybrid scoring
